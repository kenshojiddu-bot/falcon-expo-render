# 猎鹰展统一报名后台设计

## 目标

提供一个受密码保护的管理后台，统一查看猎鹰展参展申请与“精品东方·鹰耀出海”沙龙报名，并支持搜索、筛选、查看详情和导出 CSV。

## 安全

- 后台地址为 `/admin`，登录密码从 Render 环境变量 `ADMIN_PASSWORD` 读取，不写入仓库。
- 初始密码仅配置在 Render 环境变量中，不在仓库、测试或前端源码中保留明文。
- 登录成功后签发 HttpOnly、SameSite=Strict、最长 8 小时的签名 Cookie。
- Cookie 签名密钥由 `ADMIN_SESSION_SECRET` 环境变量提供。
- 同一 IP 在 15 分钟内最多允许 5 次失败登录；成功登录后清空失败记录。
- 管理 API 未登录时统一返回 401；未配置环境变量时返回 503。

## 数据

- 参展申请读取 `falcon_applications`。
- 沙龙报名读取 `salon_registrations`。
- Render 配置 Neon 时读取 PostgreSQL；本地无数据库时读取现有 JSON 文件。
- 返回统一结构并按提交时间倒序排列，不修改现有表单提交接口和数据表。

## 页面

- `/admin` 首屏为密码登录。
- 登录后显示总报名、参展申请、沙龙报名三个统计数。
- 支持“全部 / 参展申请 / 沙龙报名”筛选、关键词搜索和刷新。
- 桌面端使用紧凑表格，手机端切换为可点击列表。
- 点击记录打开详情弹窗，展示该类报名的所有业务字段。
- CSV 导出使用当前筛选和搜索结果。
- 提供退出登录，不提供编辑和删除。

## 验收

- 未登录无法读取报名数据。
- 错误密码被拒绝并计入限流，正确密码能登录和退出。
- 两类报名均能在同一后台加载、筛选、搜索、查看详情和导出。
- 页面适配 390px 手机宽度且不横向溢出。
- 现有参展申请和沙龙报名提交流程测试继续通过。
